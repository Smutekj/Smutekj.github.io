Batch Renderer

Demo

Code

A 2D batch renderer using OpenGL ES 2 (and some features of ES 3) in order to make it possible to build into WebAssembly.

Features

  • Batch rendering of sprites and vertex arrays. Batching is done using a combination of: Shader x 2 Textures
  • Static and dynamic draw calls
  • Text rendering based on FreeType2 using character SDFs makes it easy to add effects like borders to text.
  • Can choose data format of textures, so it is possible to use HDR.
  • Framebuffers and offscreen rendering

Constrained Delaunay Triangulation

Demo

Code

A constrained version of the Delaunay triangulation based on the article of Sloan. Includes option to choose type of position vector including small types, like unsigned char. This can be useful for optimisation in RTS games for instance, where maps tend to lie on not too large grids. (For example, Starcraft 2 uses 2 bytes per Vertex coordinate).

Applications:

  • Pathfinding
  • Vision polygon construction
  • Terrain construction
  • Discretisation of the physical domain in FEM/FVM simulations
  • Maybe as a step in construction of more optimal NavMeshes?

Pathfinding

Demo

Code

A shooter game with lua scripting. Not sure what I want to make out of it, yet. Graphics, collision system is done using an in-house renderer. Broad phase of the collision detection algorithm is accelerated using the Bounding Volume Hiearchy spatial data structure. (BVH implementation is based on Erin Catto's presentation. God bless Erin Catto)

Pathfinding process can be divided into two phases: Planning phase and Smoothing phase.
During the planning phase, an A* algorithm is run on a triangulation graph, which produces a sequence of triangles connecting the start and finnish points of the PathFinding (the sequence of triangles is called Funnel.)
The Funnel is then processed in the smoothing phase resulting in the final path.
A demonstration of the smoothing part of the algorithm.
